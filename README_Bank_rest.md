# 🏦 Банковская система управления картами

Проект представляет собой RESTful API для управления банковскими картами пользователей. Поддерживает регистрацию, авторизацию, переводы между картами, блокировку/активацию, шифрование номеров карт и автоматическую блокировку просроченных карт.

---

## 🛠️ Технологии

- **Java 17**
- **Spring Boot 3**
- **PostgreSQL** (в Docker)
- **Hibernate / JPA**
- **Liquibase** — миграции базы данных
- **JWT** — аутентификация и авторизация
- **Jasypt** — шифрование конфиденциальных данных
- **Docker & Docker Compose**
- **Swagger UI** — документация API
- **Validation**, **Security (Spring Security)**

---

## 📦 Функциональность

- ✅ Регистрация и вход пользователей
- ✅ Управление ролями: `USER`, `ADMIN`
- ✅ Создание, просмотр и удаление карт
- ✅ Перевод средств между картами
- ✅ Блокировка и активация карт
- ✅ Автоматическая блокировка просроченных карт (ежедневно)
- ✅ Поиск карт по параметрам (для администратора)
- ✅ Шифрование номеров карт в БД
- ✅ Пагинация и фильтрация

---

## ▶️ Запуск приложения

### 1. Клонируйте репозиторий
> bash git clone https://github.com/moratti72r/bank_rest.git   
> cd bank-cards-system

### 2. Соберите и запустите с помощью Docker Compose
bash docker-compose up --build

> При первом запуске:
> - Собирается JAR-файл приложения.
> - Запускается PostgreSQL.
> - Применяются миграции Liquibase.
> - Приложение стартует на порту `8080`.

---

### 🔐 Администратор по умолчанию

При первом запуске приложения автоматически создаётся пользователь с правами администратора.

| Поле       | Значение                                  |
|------------|-------------------------------------------|
| **Email**  | `admin@example.com`                       |
| **Пароль** | `admin123`                                |
| **Роль**   | `ROLE_ADMIN`                              |
| **UUID**   | `a0eebc99-9c0b-4ef8-bb6d-6bb9bd380a11`    |

### Как войти?

Отправьте запрос на авторизацию:

`http POST /api/auth/login Content-Type: application/json
{ "email": "admin@example.com", "password": "admin123" }`

>
> Добавьте его в заголовок последующих запросов:
>
> `Authorization: Bearer <ваш_токен>`

## 🔒 Безопасность

- Пароли хешируются через `BCrypt`.
- Номера карт шифруются с помощью **Jasypt**.
- Конфиденциальные данные не хранятся в открытом виде.
- JWT используется для аутентификации.
- Черный список токенов при выходе (logout).

---

## 🔄 Автоматические задачи

- **Ежедневная проверка просроченных карт:**

cron 0 0 0 * * ? → blockExpiredCards()
Все активные карты с датой окончания раньше сегодняшней будут переведены в статус `EXPIRED`.
